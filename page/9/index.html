R Debug moving to final location failed

Posted on 2020-09-27 | In R语言学习

原因：

The current implementation of package installation by default backs up the old installation of the package by moving it into a per-library 00LOCK directory (or per-package 00LOCK-pkgname). The installation is performed directly into the final directory pkgname in the library. If it fails, it is by default cleaned up and the old version is moved back; otherwise, if it succeeds, the old version is deleted. If the lock directory already exists when the installation is requested, the installation fails with an error and one typically would delete the directory manually. During parallel install, the per package locking is used (00LOCK-pkgname). This locking mechanism works for backing-up and recovering previous versions of packages in case of error, but it does not prevent access to partially installed packages

默认情况下，软件包安装的实现通过将软件包的旧安装移动到每个库00LOCK目录（或每个软件包00LOCK-pkgname）中来备份该软件包的旧安装。安装直接在库中的最终目录pkgname中执行。
如果失败，则默认情况下将其清除，并将旧版本移回；如果成功，将删除旧版本。
如果在请求安装时锁定目录已经存在，则安装将失败并显示错误，并且通常会手动删除该目录。
在并行安装期间，将使用每个软件包锁定（00LOCK-pkgname）。
此锁定机制适用于在发生错误的情况下备份和恢复软件包的先前版本，但是它不能防止访问部分安装的软件包
